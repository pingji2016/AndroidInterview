# 第二阶段计划（第 5-8 周）：项目实战 + 作品产出

## 整体目标

> **用 4 周做出一个能展示、能讲深、能体现差异化的端侧 AI Agent Demo，同时补一轮性能优化实战。**

### 与第一阶段的关系

- 第一阶段你砸实了 Android 基础（协程、Handler、RecyclerView、性能优化、Framework）
- 第二阶段**把这些知识用起来**，做一个完整的、可演示的项目
- 面试官看到你的简历时会想："这个人不光会八股文，还能动手做项目"

### 核心产出

| 项目 | 类型 | 产出 |
|:---|:---|:---|
| **端侧 AI Agent** | 主项目 | GitHub 仓库 + 2 分钟录屏 + 架构图 |
| **性能优化报告** | 副项目 | PDF 报告（启动/内存/卡顿分析） |

### 每天投入

- **工作日**：2 小时（代码为主）
- **周末**：6 小时（整块时间调试 + 写文档）

## 几点提醒

1. **先跑通，再优化**：第 5-6 周的核心目标是"能跑通"，不要过早追求推理速度、UI 美观
2. **如果 llama.cpp 编译卡住**：果断换 MLCEngine，不浪费时间。最后效果一样
3. **模型选择**：优先 TinyLlama Q4（~400MB），手机存储不够换 Q3（~280MB），再不够换 Braindler（88MB）
4. **录屏是重点**：面试官不会在你手机上跑代码，视频是你项目"存在"的证据
5. **每天 2 小时写代码**：第二阶段主要是动手，少看博客多写代码
6. **周末是黄金时间**：调试、联调、录视频这些需要整块时间，留到周末做

> 第二阶段结束，你将拥有一个完整的端侧 AI Agent 项目 + 一份性能优化报告。这是你简历上最硬的干货，也是面试中能讲 30 分钟的核心话题。
