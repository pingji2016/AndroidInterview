## 一、什么是 llama.cpp？

llama.cpp 是一个纯 C/C++ 实现的 LLM 推理引擎，最大的特点是**不依赖 GPU 也能跑**，专为 CPU 环境优化。

| 特点 | 说明 |
|:---|:---|
| **纯 CPU 推理** | 无需 CUDA / OpenCL，手机 CPU 即可运行 |
| **支持 GGUF 格式** | 一种专为 CPU 推理设计的模型文件格式 |
| **跨平台** | 支持 Linux、macOS、Windows、Android |
| **量化支持** | 支持 Q2～Q8 多种量化级别，控制模型大小 |

### 为什么选 llama.cpp 做端侧 Agent？

- Android 端推理的首选方案，社区活跃、文档完善
- 官方提供了 Android 示例 `examples/android/`，可直接编译运行
- 模型文件小（Q4 量化后约 400MB），手机存储可接受

## 二、环境准备与 NDK 检查

### NDK 的作用

Android NDK（Native Development Kit）允许你用 C/C++ 编写代码并编译为原生库（`.so`）。llama.cpp 的核心推理引擎就是 C++ 实现的，所以必须通过 NDK 来交叉编译出 ARM64 架构的 `.so` 文件。

### 检查 NDK 是否已安装

```bash
# 在 Android Studio 中查看 NDK 路径
# Settings → Appearance & Behavior → System Settings → Android SDK → SDK Tools
# 确保 NDK（Side by side）已勾选，推荐版本 r25c 或 r26

# 命令行验证
ndk-build --version
# 输出示例：GNU Make 4.2.1
# 或检查 NDK 路径
echo $ANDROID_NDK_HOME
# /Users/xxx/Library/Android/sdk/ndk/26.1.10909125
```

### 如果没有 NDK，安装方式

```bash
# 方式一：Android Studio SDK Manager 安装
# 方式二：sdkmanager 命令行
sdkmanager --install "ndk;26.1.10909125"
```

```
Java (Kotlin) 层
```

## 四、编译 llama.cpp Android 版本

### build_android.sh 做了什么？

```bash
#!/bin/bash
# 该脚本做了三件事：
# 1. 设置 NDK 交叉编译工具链
# 2. 用 CMake 配置 ARM64 交叉编译
# 3. 编译出 libllama.so

# 关键编译参数
cmake -DCMAKE_TOOLCHAIN_FILE=$NDK/build/cmake/android.toolchain.cmake \
      -DANDROID_ABI=arm64-v8a \           # ARM64 架构
      -DANDROID_PLATFORM=android-23 \      # 最低 API 23
      -DCMAKE_BUILD_TYPE=Release \          # Release 模式
      -DLLAMA_STATIC=OFF \                  # 编译为动态库 .so
      ..
```

### 执行编译

```bash
# 确保在 llama.cpp 根目录
cd llama.cpp

# 运行 Android 编译脚本
./scripts/build_android.sh

# 编译成功后在 build-android/ 目录下得到
ls build-android/src/
# libllama.a          # 静态库
# libllama.so         # 动态库（我们需要这个）

# 编译 Android 示例 App
./examples/android/build_android.sh
```

### 如果编译失败，常见原因

| 错误 | 原因 | 解决 |
|:---|:---|:---|
| `NDK not found` | 未设置 NDK 路径 | `export ANDROID_NDK_HOME=/path/to/ndk` |
| `cmake: command not found` | 未安装 CMake | `brew install cmake` 或 `apt install cmake` |
| `arm64-v8a not supported` | NDK 版本过低 | 升级到 NDK r25+ |
| `Build failed: No rule to make target` | 依赖缺失 | 检查是否 clone 完整（含 submodule） |

## 备注：NDK 版本选择建议

| NDK 版本 | 建议 |
|:---|:---|
| r25c (25.2.9519653) | ✅ 稳定，推荐 |
| r26 (26.1.10909125) | ✅ 兼容性好 |
| r23 及以下 | ❌ 不支持 Android 14 以上 |

推荐使用 **NDK r26**，对 ARM64 的优化更好。
