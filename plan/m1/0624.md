## 一、GGUF 模型文件格式

### 什么是 GGUF？

GGUF（GPT-Generated Unified Format）是 llama.cpp 社区定义的模型文件格式，专为 CPU 推理优化：

- **自包含**：一个 `.gguf` 文件包含模型权重、分词器、配置元数据，无需额外文件
- **支持量化**：内置多种量化方案（Q2～Q8），可大幅减小模型体积
- **兼容性好**：所有基于 llama.cpp 的推理引擎都支持

### 量化等级对比

| 量化 | 模型大小（1B 模型） | 推理速度 | 质量损失 |
|:---|:---|:---|:---|
| Q8_0 | ~1.1GB | 快 | 几乎无损 |
| Q4_K_M | ~400MB | ✅ 最快 | 轻微损失（推荐） |
| Q3_K_M | ~280MB | 快 | 明显损失 |
| Q2_K | ~180MB | 快 | 严重损失 |

**推荐 TinyLlama Q4_K_M**：体积适中（~400MB），质量损失可忽略。

### 从 Hugging Face 下载

```bash
# 方式一：直接用浏览器下载（推荐）

huggingface-cli download TheBloke/TinyLlama-1.1B-Chat-v1.0-GGUF tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf --local-dir .
```

## 三、将模型放入 Android 项目

### 方案一：放入 `assets/` 目录（推荐，最简单）

```bash
# 将模型文件复制到 llama.cpp 示例项目的 assets 目录
cp tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf llama.cpp/examples/android/app/src/main/assets/

# 确认文件已放入
ls -lh llama.cpp/examples/android/app/src/main/assets/
# tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf (410 MB)
```

### 方案二：放入手机 SD 卡（避免 APK 过大）

如果模型放 assets 会导致 APK 过大，可以用 adb push 到手机：

```bash
# 把模型推到手机存储
adb push tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf /sdcard/

# 然后在代码中从 /sdcard/ 路径加载
# llama.loadModel("/sdcard/tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf")
```

### assets 目录的限制

| 限制 | 说明 |
|:---|:---|
| APK 大小 | Google Play 要求 APK ≤ 150MB（但 debug 包无限制） |
| 加载方式 | assets 中的文件通过 `AssetManager` 读取，需要先复制到内部存储 |
| 内存占用 | 模型加载时会映射到内存，约等于模型文件大小的 1.2-1.5 倍 |

## 四、运行示例 App

### 编译示例 App

```bash
# 如果昨天已经编译过，直接重新构建 APK
cd llama.cpp/examples/android
./gradlew assembleDebug

# APK 输出位置
ls app/build/outputs/apk/debug/
# app-debug.apk
```

### 安装到手机

```bash
# 连接手机（确保 USB 调试已开启）
adb devices
# List of devices attached
# 0123456789ABCDEF    device

# 安装 APK
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 在手机上运行

```
1. 在手机上找到 "llama" App（或者 "llama.cpp"）并打开
2. 首次启动会解压模型到内部存储（约 10-20 秒）
3. 等待模型加载完成（进度条 100%）
4. 在输入框输入 "Hello" → 点击发送
5. 等待模型推理输出文字（1-2 秒）
```

### 预期效果

```
输入: "Hello"
输出: "Hello! How can I help you today?"

输入: "What is Android?"
输出: "Android is a mobile operating system developed by Google..."
```

### 如果 App 闪退或崩溃

| 症状 | 原因 | 解决 |
|:---|:---|:---|
| 启动就闪退 | 模型路径错误 | 确认 assets 中的 .gguf 文件路径 |
| 加载到 80% 闪退 | 内存不足 | 换 Q3 量化的模型，或关闭后台 App |
| 推理不出文字 | 模型与示例不兼容 | 确认下载的是 Chat 版本，不是 Base 版本 |
| 推理极慢（>30s） | 手机 CPU 太弱或未开启多线程 | 设置 `-t 4` 使用 4 线程 |
