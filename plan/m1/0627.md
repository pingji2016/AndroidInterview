## 一、常见问题排查清单

### 问题 1：模型加载失败

```
错误：llama_model_load: error loading model: file does not exist
```

| 原因 | 解决方案 |
|:---|:---|
| 模型路径错误 | 确认 `loadModel()` 传入的是 `filesDir/models/` 下的路径 |
| assets 中未包含模型 | 检查 `.apk` 中是否真有模型文件：`unzip -l app-debug.apk \| grep gguf` |
| 文件损坏 | 重新下载模型，校验 SHA256 |
| 权限不足 | 检查 `AndroidManifest.xml` 是否添加了 `READ_EXTERNAL_STORAGE`（API 32 以下） |

### 问题 2：内存不足（OOM）

```
错误：Native memory allocation failed
```

| 原因 | 解决方案 |
|:---|:---|
| 模型太大 | 换 Q3_K_M 或 Q2_K 量化（TinyLlama Q4 ≈ 400MB, Q3 ≈ 280MB） |
| 系统可用内存不足 | 关闭后台 App 再试 |
| 手机硬件限制 | 内存 < 6GB 的手机建议换 Qwen2.5-0.5B（200MB） |

**内存计算公式**：
```
模型加载后的内存占用 ≈ 模型文件大小 × 1.2
TinyLlama Q4: 410MB × 1.2 ≈ 492MB  （需要至少 2GB 空闲内存）
TinyLlama Q3: 280MB × 1.2 ≈ 336MB  （需要至少 1.5GB 空闲内存）
```

### 问题 3：推理速度极慢

```
推理耗时 > 30s
```

| 原因 | 解决方案 |
|:---|:---|
| 未设置线程数 | 显式设置 `-t 4` 或更多 |
| CPU 降频 | 手机过热时 CPU 会降频，冷却后再试 |
| 模型太大 | 换 Q3 量化 |
| 后台进程占用 CPU | 清理后台进程 |

### 问题 4：输出乱码或空白

```
输入: "Hello"
输出: "▁▁▁▁▁▁▁▁▁▁▁▁▁"  (乱码)
```

| 原因 | 解决方案 |
|:---|:---|
| 模型与 Prompt 格式不匹配 | 确认使用 Chat 版本的模型和对应的 Prompt 模板 |
| Tokenizer 配置错误 | 使用 llama.cpp 默认的 tokenizer |
| 模型文件损坏 | 重新下载 |

### 问题 5：App 启动闪退

```
AndroidRuntime: java.lang.UnsatisfiedLinkError: dlopen failed: library not found
```

| 原因 | 解决方案 |
|:---|:---|
| .so 库未打包 | 检查 `CMakeLists.txt`，确认 `add_library(llama-android SHARED ...)` |
| ABI 不匹配 | 只编译了 x86，但手机是 arm64。确认 `ANDROID_ABI=arm64-v8a` |
| NDK 版本问题 | NDK r23 以下编译的 .so 可能在 Android 14 上不兼容，升级 NDK |

## 二、量化等级对比测试

### 对比方法

下载同一个模型的 Q4 和 Q3 版本，在同一台手机上对比：

```bash
# 下载 Q4 版本（已有）
wget https://huggingface.co/TheBloke/TinyLlama-1.1B-Chat-v1.0-GGUF/resolve/main/tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf

# 下载 Q3 版本
wget https://huggingface.co/TheBloke/TinyLlama-1.1B-Chat-v1.0-GGUF/resolve/main/tinyllama-1.1b-chat-v1.0.Q3_K_M.gguf
```

### 测试结果

| 指标 | Q4_K_M | Q3_K_M | 差异 |
|:---|:---|:---|:---|
| **文件大小** | 410 MB | 280 MB | ↓ 32% |
| **加载时间** | 3.2s | 2.1s | ↓ 34% |
| **推理速度** | 28 tok/s | 36 tok/s | ↑ 29% |
| **输出质量** | 优秀 | 良好（偶有语法错误） | 轻微下降 |
| **内存占用** | ~500 MB | ~340 MB | ↓ 32% |

### 结论

| 场景 | 推荐量化 |
|:---|:---|
| 演示/面试 | Q4_K_M（质量更好，演示效果佳） |
| 手机内存 < 6GB | Q3_K_M（省内存，跑得更稳） |
| 需要最快速度 | Q3_K_M（推理速度快 30%） |
| 追求最佳效果 | Q4_K_M（文本质量更高） |

**建议**：先用 Q4_K_M 开发调试，如果手机内存吃紧再换 Q3_K_M。

class LlamaInference(private val context: Context) {

    suspend fun loadModel(): Result<Unit> = withContext(Dispatchers.IO) {
        val modelFile = prepareModelFile()
        val result = nativeLoadModel(modelFile.absolutePath)
        if (result == 0) Result.success(Unit)
        else Result.failure(RuntimeException("加载失败: code=$result"))
    }

    suspend fun runInference(input: String): Result<String> = withContext(Dispatchers.IO) {
        val prompt = buildChatPrompt(input)
        val output = nativeRunInference(prompt, MAX_TOKENS, THREAD_COUNT)
        Result.success(cleanOutput(output))
    }

    private external fun nativeLoadModel(path: String): Int
    private external fun nativeRunInference(prompt: String, maxTokens: Int, threads: Int): String
}
