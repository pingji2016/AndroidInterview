## 二、复习回调：协程核心要点（口述用）

### suspend 函数原理

```
suspend fun 被编译后 →
  1. 多一个 Continuation 参数（回调接口）
  2. 方法体变成状态机（label 标签）
  3. 挂起点对应一个 label，恢复时跳转到对应 label

关键：
  - 挂起时释放线程，不阻塞
  - 恢复时在线程池中继续执行
  - 所以协程本质是"回调 + 状态机"的语法糖
```

### Handler 机制概览

```
Looper.prepare() → 创建 Looper + MessageQueue
Looper.loop() → 死循环取消息
  ├─ MessageQueue.next()   ← 阻塞等待（nativePollOnce）
  ├─ msg.target.dispatchMessage(msg)  ← 分发
  └─ msg.recycle()         ← 回收复用

Message 复用：
  - Message.obtain() 从对象池取（sPool 链表）
  - 最大缓存 50 个
  - 避免频繁创建对象
```

## 三、ReAct 循环笔记（预习用）

### 什么是 ReAct？

ReAct = **Rea**soning + **Act**ing，由 Google 在 2022 年提出（论文 *"ReAct: Synergizing Reasoning and Acting in Language Models"*）。

### 核心思路

```
传统 LLM 调用：
  用户输入 → LLM 输出文字 → 结束

ReAct 循环：
  用户输入 → LLM 思考(Thought) → 决定行动(Action)
    → 执行工具(Action Input) → 观察结果(Observation)
    → LLM 再次思考 → ... → 最终回答(Final Answer)
```

### ReAct 循环伪代码

```python
# ReAct 循环的简化伪代码
def react_loop(user_input, tools, max_steps=5):
    # 构建初始 prompt
    prompt = build_prompt(user_input, tools)
    
    for step in range(max_steps):
        # 1. LLM 思考
        response = llm.inference(prompt)
        
        # 2. 解析输出
        thought = extract_thought(response)
        action = extract_action(response)  # 如: "sendNotification"
        action_input = extract_action_input(response)  # 如: "任务完成"
        
        if action == "Final Answer":
            # 3. 最终回答
            return action_input
        
        # 4. 执行工具
        result = tools.execute(action, action_input)
        
        # 5. 将观察结果加入 prompt
        prompt += f"Observation: {result}\nThought: "
    
    return "Max steps reached"
```

fun reactLoop(userInput: String): String {
    var prompt = buildReActPrompt(userInput)
    
    repeat(MAX_STEPS) {
        // 1. LLM 推理
        val output = llm.runInference(prompt)

            output.contains("Action: sendNotification") -> {

                val level = extractParam(output, "level")
                brightnessTool.execute(level)
                prompt += "Observation: 亮度已调整\n"
            }
            output.contains("Final Answer:") -> {
                return extractFinalAnswer(output)
            }
        }
    }
