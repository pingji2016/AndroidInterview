### 常见启动瓶颈排查

```bash
瓶颈 1: ContentProvider 太多
  每个 ContentProvider 都会在 Application.onCreate 之前初始化
  检查 AndroidManifest.xml 中的 <provider> 数量
  
瓶颈 2: Application.onCreate 中做太多事
  避免：初始化 SDK、创建数据库、加载偏好设置
  优化：懒加载、推迟到首次使用时初始化

瓶颈 3: Activity 布局过于复杂
  检查布局层级（Layout Inspector）
  优化：减少嵌套、使用 ConstraintLayout、ViewStub 延迟加载

瓶颈 4: 首屏网络请求阻塞
  优化：使用本地缓存、启动时显示缓存数据
```

## 二、内存分析

### Memory Profiler 使用

```bash
# 1. 在 Android Studio 中打开 Memory Profiler
  View → Tool Windows → Profiler
  选择设备 → 选择 Now in Android 进程
  
# 2. 记录内存使用
  点击 "Record" 开始记录
  操作 App：首页 → 进入文章 → 返回 → 书签 → 搜索
  点击 "Stop" 结束记录

# 3. 关键指标
  Java Heap: 当前 Java 堆内存使用量（MB）
  Native Heap: Native 层内存（通常是图像/资源）
  Code: 代码和资源（DEX/So 文件）
  Stack: 栈内存（线程数 × 栈大小）
  Graphics: 图形缓冲区（SurfaceFlinger）
```

### 常见内存泄漏场景

```bash
场景 1: 静态引用 Activity
  class Utils { companion object { var context: Context? = null } }
  → Activity 退出后无法被 GC 回收
  → 修复：使用 ApplicationContext

场景 2: 内部类持有外部引用
  class MyActivity {
    val handler = object : Handler() {
      override fun handleMessage(msg: Message) { ... }
    }
  }
  → Handler 持有 Activity 引用，Looper 线程存活时无法回收
  → 修复：使用 static 内部类 + WeakReference

场景 3: 匿名内部类/回调
  viewModel.loadData(object : Callback {
    override fun onResult(data: Data) { updateUI(data) }
  })
  → 回调持有 Activity 引用，ViewModel 生命周期比 Activity 长
  → 修复：在 Activity onDestroy 中取消回调

场景 4: 未取消的协程
  viewModelScope.launch {
    while(true) { delay(1000); updateUI() }
  }
  → ViewModel 退出后协程未取消，更新已销毁的 UI
  → 修复：使用 lifecycleScope 或手动取消
```

### MAT 分析 Heap Dump

```bash
# 1. 导出 Heap Dump
  Android Studio Profiler → 点击 "Dump Java Heap"
  或者：adb shell am dumpheap <pid> /data/local/tmp/heap.hprof

# 2. 转换格式（Android hprof → MAT hprof）
  hprof-conv heap.hprof mat.hprof

# 3. 在 MAT 中分析
  File → Open Heap Dump → 选择 mat.hprof

# 4. 分析步骤
  Step 1: Histogram → 按类名查看实例数量
    → 搜索 MainActivity，看 instance count
    → 正常：1（当前显示），异常：>1（泄漏）
    
  Step 2: Dominator Tree → 按保留堆大小排序
    → 找到最大的对象
    → 分析它为什么存活
  
  Step 3: GC Roots → 查看引用链
    → 右键对象 → Merge Shortest Paths to GC Roots
    → 排除软/弱引用（exclude all phantom/weak/soft references）
    → 剩下的就是阻止 GC 的强引用链
```

## 三、卡顿分析

### FrameMetrics 原理

```bash
Android 每 16ms 发出一次 vsync 信号
如果一帧的渲染时间 > 16ms，就会掉帧（jank）

帧生命周期的阶段：
  1. Input Handling     → 输入事件处理
  2. Animation          → 动画计算
  3. Layout & Measure   → 布局测量
  4. Draw               → 绘制
  5. Sync & Upload      → 同步到 GPU
  6. GPU Draw           → GPU 渲染

哪个阶段超时 → 哪个阶段就是瓶颈
```

### 使用 FrameMetrics 监听

```kotlin
// 在 Activity 中注册 FrameMetrics 监听
class MainActivity : AppCompatActivity() {

    private lateinit var frameMetrics: FrameMetricsManager

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        // 注册 FrameMetrics 监听
        window.addOnFrameMetricsAvailableListener(
            { _, frameMetrics, _ ->
                // frameMetrics 是 FrameMetrics 对象
                val totalDuration = frameMetrics.getMetric(
                    FrameMetrics.TOTAL_DURATION
                ) // 总耗时（纳秒）
                
                val inputHandling = frameMetrics.getMetric(
                    FrameMetrics.INPUT_HANDLING_DURATION
                )
                
                val layoutMeasure = frameMetrics.getMetric(
                    FrameMetrics.LAYOUT_MEASURE_DURATION
                )
                
                val drawDuration = frameMetrics.getMetric(
                    FrameMetrics.DRAW_DURATION
                )
                
                // 转成毫秒
                val totalMs = totalDuration / 1_000_000f
                
                if (totalMs > 16f) {
                    Log.w("FrameMetrics", "掉帧: ${totalMs}ms")
                    Log.w("FrameMetrics", "  Input: ${inputHandling / 1_000_000f}ms")
                    Log.w("FrameMetrics", "  Layout: ${layoutMeasure / 1_000_000f}ms")
                    Log.w("FrameMetrics", "  Draw: ${drawDuration / 1_000_000f}ms")
                } else {
                    Log.d("FrameMetrics", "帧正常: ${totalMs}ms")
                }
            },
            Handler(Looper.getMainLooper())
        )
    }

    override fun onDestroy() {
        super.onDestroy()
        window.removeOnFrameMetricsAvailableListener { }
    }
}
```

### 帧率统计工具类

```kotlin
// app/src/main/java/com/example/androidagent/performance/FrameStatsCollector.kt

package com.example.androidagent.performance

import android.view.FrameMetrics
import android.view.Window
import android.os.Handler
import android.os.Looper
import java.util.concurrent.ConcurrentLinkedQueue

/**
 * 帧率统计收集器
 *
 * 功能：
 * 1. 收集每帧的耗时
 * 2. 统计掉帧率
 * 3. 按阶段分析瓶颈
 */
class FrameStatsCollector {

    private val frames = ConcurrentLinkedQueue<FrameData>()
    private var isCollecting = false

    fun start(window: Window) {
        isCollecting = true
        window.addOnFrameMetricsAvailableListener(
            { _, metrics, _ ->
                if (!isCollecting) return@addOnFrameMetricsAvailableListener
                
                val totalNs = metrics.getMetric(FrameMetrics.TOTAL_DURATION)
                val totalMs = totalNs / 1_000_000f
                val isJank = totalMs > 16f
                
                frames.add(
                    FrameData(
                        totalMs = totalMs,
                        isJank = isJank,
                        inputMs = metrics.getMetric(FrameMetrics.INPUT_HANDLING_DURATION) / 1_000_000f,
                        layoutMs = metrics.getMetric(FrameMetrics.LAYOUT_MEASURE_DURATION) / 1_000_000f,
                        drawMs = metrics.getMetric(FrameMetrics.DRAW_DURATION) / 1_000_000f,
                        gpuMs = metrics.getMetric(FrameMetrics.GPU_DURATION) / 1_000_000f,
                        timestamp = System.currentTimeMillis()
                    )
                )
            },
            Handler(Looper.getMainLooper())
        )
    }

    fun stop(window: Window) {
        isCollecting = false
        window.removeOnFrameMetricsAvailableListener { }
    }

    fun generateReport(): FrameReport {
        val totalFrames = frames.size
        val jankFrames = frames.count { it.isJank }
        val avgMs = frames.takeLast(100).map { it.totalMs }.average()

        val totalInput = frames.sumOf { it.inputMs.toDouble() }
        val totalLayout = frames.sumOf { it.layoutMs.toDouble() }
        val totalDraw = frames.sumOf { it.drawMs.toDouble() }
        val totalGpu = frames.sumOf { it.gpuMs.toDouble() }

        return FrameReport(
            totalFrames = totalFrames,
            jankFrames = jankFrames,
            jankRate = if (totalFrames > 0) jankFrames.toFloat() / totalFrames else 0f,
            averageFrameTimeMs = avgMs,
            averageInputMs = totalInput / totalFrames,
            averageLayoutMs = totalLayout / totalFrames,
            averageDrawMs = totalDraw / totalFrames,
            averageGpuMs = totalGpu / totalFrames
        )
    }

    data class FrameData(
        val totalMs: Float,
        val isJank: Boolean,
        val inputMs: Float,
        val layoutMs: Float,
        val drawMs: Float,
        val gpuMs: Float,
        val timestamp: Long
    )

    data class FrameReport(
        val totalFrames: Int,
        val jankFrames: Int,
        val jankRate: Float,
        val averageFrameTimeMs: Double,
        val averageInputMs: Double,
        val averageLayoutMs: Double,
        val averageDrawMs: Double,
        val averageGpuMs: Double
    )
}
```
