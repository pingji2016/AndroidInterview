## 二、最终追问答案清单

```
6 个问题的核心话术提炼（每句 10 秒）:

Q1: 为什么端侧？
    "低延迟、隐私保护、离线可用，适合手机场景。"

Q2: 模型怎么选？
    "1B 是手机能接受的极限，Q4_K_M 量化和质量的最优平衡。"

Q3: 延迟多少？
    "加载 3s，单步推理 2.5s，两步任务约 5s。
     已优化：多线程、限制输出长度、后台预加载。"

Q4: JSON 解析失败怎么办？
    "先正则提取 JSON 块，失败则降级为关键词匹配。
     连续 3 次失败提示用户换种方式。"

Q5: 跟 LangChain 的区别？
    "核心思路一致（Tool + ReAct），但面向 Android 端，
     内存受限，集成的是系统 API 而非网络 API。"

Q6: Tool 接口设计？
    "AgentTool 接口 + ToolManager 注册表，
     开闭原则，新增工具只需实现接口。"
```

## 三、第一阶段笔记最终回顾

### 快速回顾方法

```bash
# 对着每个知识点，尝试 30 秒内说出核心要点
# 说不出来的 → 翻笔记 → 再口述一遍

□ 进程/线程/协程
  核心: 协程 = 状态机 + Continuation，挂起不阻塞线程

□ Activity 启动模式
  核心: standard/singleTop/singleTask/singleInstance

□ View 事件分发
  核心: dispatchTouchEvent → onInterceptTouchEvent → onTouchEvent

□ RecyclerView 缓存
  核心: 四级缓存 (Scrap/Cache/ViewPool/Extension)

□ Handler 机制
  核心: Looper + MessageQueue + Message.obtain 复用

□ 启动优化
  核心: Perfetto 分析 → 找出耗时 → 异步/懒加载

□ 内存泄漏
  核心: LeakCanary (WeakReference + ReferenceQueue)

□ AMS 启动
  核心: Launcher → AMS → Zygote(Socket) → ActivityThread

□ WMS 窗口管理
  核心: Window → WindowManager → WMS(Binder) → SurfaceFlinger

□ Input 事件分发
  核心: InputReader → InputDispatcher → ViewRootImpl → View

□ ANR
  核心: Input 5s / Broadcast 10s / Service 20s

□ Binder
  核心: 一次拷贝、C/S 架构、ServiceManager 注册发现

□ SurfaceFlinger
  核心: Surface(生产者) → BufferQueue → SurfaceFlinger(消费者) → HWC
```

### 如果发现某个知识点不熟练

```bash
1. 立即找到对应的 m1 笔记文件
2. 快速通读一遍（3-5 分钟）
3. 用自己的话录音叙述（2 分钟）
4. 第二天再复习一次
```
