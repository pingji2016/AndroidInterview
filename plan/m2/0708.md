### 博客关键代码段

```kotlin
// 接口定义
interface AgentTool {
    fun getSchema(): ToolSchema
    suspend fun execute(params: Map<String, Any>): ToolResult
}

// Tool Schema
data class ToolSchema(
    val name: String,
    val description: String,
    val parameters: List<ParameterSchema>
)

// ToolManager
class ToolManager {
    private val tools = mutableMapOf<String, AgentTool>()
    fun register(tool: AgentTool) { tools[tool.getSchema().name] = tool }
    suspend fun execute(action: ParsedAction): ToolResult {
        val tool = tools[action.action] ?: return ToolResult.failure("未知工具")
        return tool.execute(action.params)
    }
}
```

### 编译可能出现的问题

```bash
问题 1: Could not find com.android.tools.build:gradle:8.x
  解决: 确认 Android Studio 版本 >= Hedgehog
  检查 gradle/wrapper/gradle-wrapper.properties

问题 2: SDK location not found
  解决: 设置 ANDROID_HOME 环境变量
  或在 local.properties 中指定 sdk.dir

问题 3: Kotlin 版本冲突
  解决: ./gradlew :app:dependencies > deps.txt
  查看依赖树，找到冲突版本

问题 4: Task :app:lint 失败
  解决: ./gradlew :app:assembleDebug -x lint
  跳过 lint 检查

问题 5: 下载依赖超时
  解决: 配置阿里云镜像仓库
  maven { url 'https://maven.aliyun.com/repository/public' }
```

## 三、启动性能分析

### 冷启动 vs 热启动 vs 温启动

```bash
冷启动（Cold Start）：
  进程不存在 → 创建进程 → 初始化 Application → 创建 MainActivity
  → 测量的是从点击图标到首帧渲染的时间

温启动（Warm Start）：
  进程存在，Activity 被回收 → 重新创建 Activity
  → 比冷启动快，因为没有 Application 初始化

热启动（Hot Start）：
  进程存在，Activity 存在内存中
  → 只是 bring to front
```

### 方法 1：adb shell am start -W

```bash
# 基本命令
adb shell am start -W -n com.google.samples.apps.nowinandroid/.MainActivity

# 输出示例
Starting: Intent { act=android.intent.action.MAIN cat=[android.intent.category.LAUNCHER] ... }
Status: ok
LaunchState: COLD
Activity: com.google.samples.apps.nowinandroid/.MainActivity
TotalTime: 1248
WaitTime: 1265
Complete

# 字段说明
  TotalTime: 从启动到 Activity 显示第一帧的时间（毫秒）
  WaitTime: 包含系统调度时间的总耗时（一般比 TotalTime 大几毫秒）
  LaunchState: COLD / WARM / HOT
```

### Perfetto trace 分析要点

```bash
关键分析维度：

1. CPU 使用率
   → 查看 CPU 是否满载，是否有瓶颈
   → 关注 cpu_idle 占比

2. 主线程（MainThread）活动
   → onCreate / onStart / onResume 各阶段耗时
   → 是否有长时间阻塞（>16ms 的帧）

3. 布局渲染
   → Choreographer 帧率
   → 掉帧位置和原因

4. 磁盘 I/O
   → Database 查询耗时
   → SharedPreferences 加载

5. 网络请求
   → 如果有网络请求，查看耗时

6. GC 事件
   → GC 次数和耗时
   → 是否触发频繁 GC
```

### 具体分析步骤

```bash
Step 1: 在 Perfetto UI 中加载 trace 文件
  → 打开 https://ui.perfetto.dev
  → 点击 "Open trace file" → 选择 trace.perfetto-trace

Step 2: 定位启动时间段
  → 搜索 "StartActivity" 或 "proc/[pid]"
  → 找到应用进程创建的时间点
  → 标记为启动起点

Step 3: 分析 Application 初始化
  → 搜索 "MyApplication.onCreate" 或 "Application.onCreate"
  → 查看耗时
  → 检查 ContentProvider 数量（每个 Provider 会增加启动时间）

Step 4: 分析 Activity 创建
  → 搜索 "MainActivity.onCreate"
  → 从 setContentView 到第一帧渲染的耗时

Step 5: 标记优化点
  → 在 trace 上用红色标注耗时最长的阶段
  → 截图保存
```
