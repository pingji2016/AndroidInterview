## 一、复习回调：AMS 启动 / Input 事件 / Binder 口述要点

### AMS 启动 Activity 流程

```
点击桌面图标 → Launcher 进程
  1. Launcher.startActivity() → Instrumentation.execStartActivity()
  2. → Binder 跨进程 → ActivityManagerService.getDefault().startActivity()
  3. AMS 检查 Activity 是否已注册（PackageManager）
  4. AMS 检查进程是否存在
     ├─ 存在 → 直接发送消息给 ApplicationThread
     └─ 不存在 → 通过 Socket 通知 Zygote fork 新进程
                     → 新进程入口 ActivityThread.main()
                     → 绑定 Application
                     → 创建 Activity → onCreate/onStart/onResume
  5. AMS 执行 Activity 生命周期调度（通过 ApplicationThread Binder 回调）
  6. Activity.onResume() → 界面可见
```

### Input 事件分发流程

```
物理触摸 → 屏幕驱动 → 内核 /dev/input/eventX
  1. InputReader 线程读取事件（EventHub / epoll）
  2. InputClassifier 处理（多点触控、手势识别）
  3. InputDispatcher 线程分发
     → 找到焦点窗口（WMS 提供）
     → 跨进程通过 InputChannel（SocketPair）发送
  4. App 端的 ViewRootImpl 接收
  5. DecorView → Activity.dispatchTouchEvent()
     → PhoneWindow → super.dispatchTouchEvent()
     → ViewGroup.dispatchTouchEvent() 递归分发给子 View
     → onTouchEvent() 处理
```

### Binder 一次完整通信

```
请求方进程                    Binder 驱动                  服务方进程
    │                           │                           │
    │── transact() ──────────→   │── 映射内存 ───────────→   │
    │   (写入 Parcel)            │   (一次拷贝)               │
    │                           │                           │
    │                           │←── onTransact() ─────────│
    │←── 返回结果 ────────────   │   (回复 Parcel)            │
    │                           │                           │
    
关键：
  - 一次拷贝：用户空间 → 内核空间，服务方 mmap 共享
  - 线程管理：Binder 驱动自动维护线程池
  - 同步调用：transact() 阻塞等待回复
```

