## 一、复习：RecyclerView 缓存机制

### RecyclerView 四级缓存

```
RecyclerView 缓存 = 四级缓存结构
  Level 1: mAttachedScrap (屏幕内可见)
    存正在显示的 ViewHolder，复用时不走 onBind
    场景：item 移动、动画、刷新时直接取回
  
  Level 2: mCachedViews (屏幕外缓存)
    默认容量 2，存刚刚滑出屏幕的 ViewHolder
    复用时不走 onBind，直接复用
    场景：上滑后下滑回来的场景（命中率最高）
  
  Level 3: mViewCacheExtension (自定义缓存)
    开发者自己实现，一般不用
  
  Level 4: mRecyclerPool (共享池)
    默认每个 ViewType 容量 5
    复用必须走 onBind
    多 RecyclerView 可共享同一个 Pool
```

### 缓存命中流程

```
getViewForPosition(pos)
    │
    ├── Level 1: mAttachedScrap.get(pos) → 命中 → 直接返回 ✅
    │
    ├── Level 2: mCachedViews.get(pos) → 命中 → 直接返回 ✅
    │
    ├── Level 3: mViewCacheExtension.get(pos) → 命中 → 直接返回 ✅
    │
    ├── Level 4: mRecyclerPool.get(viewType) → 命中 → onBindViewHolder() ⚠️
    │
    └── 全部未命中 → createViewHolder() + onBindViewHolder() 🔴
```

### 高频面试题

```bash
Q1: RecyclerView 滑回时闪烁/重绘？
    → 检查是否在 Adapter 中每次都调用了 notifyDataSetChanged()
    → 应该使用 notifyItemInserted/Removed 等精确方法

Q2: 多布局 RecyclerView 优化点？
    → 善用 getItemViewType() 区分类型
    → ViewType 数量控制在合理范围（Pool 按 ViewType 缓存）

Q3: 如何预加载？
    → RecyclerView.setItemViewCacheSize(size) 扩大 Level 2 缓存
    → 配合 DiffUtil 做增量更新

Q4: 如何监控 RecyclerView 性能？
    → 使用 RecyclerView.setRecyclerListener { holder -> Log.d(...) }
    → 查看 Level 4 命中率，过低说明 Pool 不足或缺省 ViewType
```

## 二、复习：Activity 启动模式

### 四种启动模式

| 模式 | 启动方式 | 行为 | 场景 |
|:---|:---|:---|:---|
| **standard** | 默认 | 每次创建新实例 | 普通页面 |
| **singleTop** | 栈顶复用 | 栈顶时重用 onNewIntent | 通知栏跳转 |
| **singleTask** | 任务栈内唯一 | 整个栈唯一，清除上面的 | 主页 |
| **singleInstance** | 单独任务栈 | 独占一个栈 | 拨号、Launcher |

### Intent Flag 补充

```bash
FLAG_ACTIVITY_NEW_TASK       ≈ singleTask
FLAG_ACTIVITY_SINGLE_TOP     ≈ singleTop
FLAG_ACTIVITY_CLEAR_TOP      = 清除目标之上的 Activity
FLAG_ACTIVITY_NO_HISTORY     = 退出后不保留
```

### 高频场景题

```bash
Q1: App 从通知栏点击跳转，怎么保证不重复创建？
    → PendingIntent 使用 FLAG_UPDATE_CURRENT
    → Activity 设置 singleTop + onNewIntent()

Q2: 主页点返回键退出 App？
    → 标记 clearTaskOnLaunch
    → 或 ActivityManager.killBackgroundProcesses()

Q3: 打开浏览器怎么独立栈？
    → 浏览器 Activity 设 singleInstance
    → 独立任务栈，不干扰 App 栈
```

### 错误重试机制

```kotlin
// 在 ChatViewModel 中添加重试
fun retry() {
    val lastUserMessage = _messages.value.lastOrNull { it.isUser }
        ?: return
    
    // 移除最后一条错误消息
    _messages.value = _messages.value.dropLastWhile { !it.isUser }
    
    // 重新发送
    sendMessage(lastUserMessage.content)
}
```

### ViewModel 更新

```kotlin
// ChatViewModel.sendMessage() 中设置状态
fun sendMessage(text: String) {
    _messages.value = _messages.value + ChatMessage(
        content = text,
        isUser = true,
        status = MessageStatus.SENDING
    )

    // 添加 thinking 占位
    _messages.value = _messages.value + ChatMessage(
        content = "",
        isUser = false,
        status = MessageStatus.THINKING
    )

    viewModelScope.launch {
        _isLoading.value = true
        val startTime = System.currentTimeMillis()

        try {
            val result = agent.execute(
                userInput = text,
                onStep = { step ->
                    // 更新为执行中
                    _messages.value = _messages.value.mapIndexed { index, msg ->
                        if (index == _messages.value.lastIndex && !msg.isUser) {
                            msg.copy(
                                content = "→ ${step.thought}\n  执行: ${step.action}",
                                status = MessageStatus.EXECUTING
                            )
                        } else msg
                    }
                }
            )

            val elapsed = System.currentTimeMillis() - startTime

            // 替换 thinking 占位为实际结果
            _messages.value = _messages.value.mapIndexed { index, msg ->
                if (index == _messages.value.lastIndex && !msg.isUser) {
                    msg.copy(
                        content = result,
                        status = MessageStatus.DONE,
                        inferenceTimeMs = elapsed
                    )
                } else msg
            }
        } catch (e: Exception) {
            _messages.value = _messages.value.mapIndexed { index, msg ->
                if (index == _messages.value.lastIndex && !msg.isUser) {
                    msg.copy(
                        content = "执行出错: ${e.message}",
                        status = MessageStatus.ERROR
                    )
                } else msg
            }
        }

        _isLoading.value = false
    }
}
```

## 四、Agent 性能优化

### 问题 1：模型加载时间长

```bash
当前情况：
  每次 App 启动 → 加载模型文件（~200MB）→ 耗时 3-5 秒
  用户打开 App 后需要等待模型加载完成才能聊天

优化方案：
  [方案 A] 懒加载：App 启动时不加载，首次聊天时加载
    → 优点：启动即用
    → 缺点：首次聊天等待更长（延迟痛苦）
    
  [方案 B] 后台预加载：Application.onCreate 中异步加载
    → 优点：用户聊天时已加载好
    → 缺点：占用启动时间
    
  [方案 C] ✅ 最优：后台预加载 + Loading 页
    → Application.onCreate 启动异步加载
    → 主页显示 Loading 进度条
    → 加载完成自动进入聊天
    → 如果加载失败，显示重试按钮
```

### 预加载实现

```kotlin
// app/src/main/java/com/example/androidagent/MyApplication.kt

class MyApplication : Application() {

    lateinit var agent: ReActAgent
        private set

    private val _isModelLoaded = MutableStateFlow(false)
    val isModelLoaded: StateFlow<Boolean> = _isModelLoaded.asStateFlow()

    override fun onCreate() {
        super.onCreate()
        
        // 1. 初始化 ToolManager（轻量，不阻塞）
        val toolManager = ToolManager().apply {
            register(SendNotificationTool(this@MyApplication))
            register(AdjustBrightnessTool(this@MyApplication))
            register(GetCurrentAppTool(this@MyApplication))
            register(SetVolumeTool(this@MyApplication))
            register(OpenAppTool(this@MyApplication))
        }
        
        // 2. 初始化推理引擎（轻量）
        val inference = LlamaInference(this)
        
        // 3. 初始化 Agent
        agent = ReActAgent(inference, toolManager)
        
        // 4. 后台预加载模型
        CoroutineScope(Dispatchers.IO).launch {
            val startTime = System.currentTimeMillis()
            Log.d("ModelLoad", "开始加载模型...")
            
            inference.loadModel()  // 耗时操作
            
            val elapsed = System.currentTimeMillis() - startTime
            Log.d("ModelLoad", "模型加载完成，耗时: ${elapsed}ms")
            
            _isModelLoaded.value = true
        }
    }
}
```

### 问题 2：多轮对话未缓存

```bash
当前情况：
  第 1 轮：用户："调亮屏幕" → Agent："已调亮" ✅
  第 2 轮：用户："再暗一点" → Agent："什么再暗一点？" ❌
  
  原因：每次 execute() 是全新 ReAct 循环
```

### 解决方案：ConversationManager

```kotlin
// app/src/main/java/com/example/androidagent/agent/ConversationManager.kt

class ConversationManager(
    private val maxHistoryTurns: Int = 3
) {
    private val history = mutableListOf<ConversationTurn>()

    fun addTurn(userInput: String, agentResponse: String) {
        history.add(ConversationTurn(userInput, agentResponse))
        if (history.size > maxHistoryTurns) {
            history.removeAt(0)
        }
    }

    fun buildContext(): String {
        if (history.isEmpty()) return ""
        return history.joinToString("\n") { turn ->
            """
用户: ${turn.userInput}
助手: ${turn.agentResponse}
            """.trimIndent()
        }
    }

    fun clear() { history.clear() }

    data class ConversationTurn(
        val userInput: String,
        val agentResponse: String
    )
}
```
